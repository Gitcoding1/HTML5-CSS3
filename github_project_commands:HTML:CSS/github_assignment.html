Git Basics

Git is a version control system; you can think of it as a series of snapshots (commits) of your code base. with the creation of each snapshot you bulid out a whole application, seeing the growth of a flll app. During this time you cna make branches to experiment and come back to a previous snapshot you took or merge those experiments into the your app.

What is GitHub?

GitHub is a code hosting paltform for version control and collaboration. It lets you and others work together on projects from anywhere using those code snapshots that you took during developmemt.

Git init

Git init is the act of registering this current code base.

Git add

Git add is how we stage the intergration of out current code bade into our current list of snapshots

Git status

git status allows us to see the changes we are going to make

Git commit

Git Commit; finally we can commit those changes we added to out lost snapshot giving us a new snapshot of our code with whatever new chandes we made

Git remote add

Git remote add is used to set up an online repository were we can place all of our current code. This is where the outside world can see out code base.

Git push

Git push is how we get our code to Github, an excellent place to store all of our repositories. Others developers might look at our existing projects to see how we approched problems, pull down our code and add new features or even update our code base with features ther wanted to add
